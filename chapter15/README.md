
### 毕业总结
```text
  作为一名Java开发，最近一年转Go，对于Docker,K8S早有耳闻。但是对于它们的了解仅仅停留在知道而已。
要全面了解一个事物要从3W1H（what,when,why,how)入手，于是乎不在乎钱包呐喊的我，毅然决然的报了这门课程

  从毕业至今，我在课程上的花费并不算少，尤其是如今处于知识付费的时代，市面上的课程，书籍可谓是浩如烟海，其中有许多散播焦虑的，滥竽充数的课程
对于技术的学习，太理论的不行，往往学过之后，十不存一，一定是要走理论实践相结合的路子

  近些年来，各个大厂在降本增效的方面的投入越来越多，到了业务发展的稳定器，这是必走的一步，云原生技术的应用，已经成为大厂在基建方面必定要用的技术，毕竟容器化的诞生就是为了解决资源利用率的问题。
到了今天已经演变成独立产品，像阿里云，腾讯云，华为云等等，各个大厂都相继推出自己的云平台。
所以，尽管作为一名开发，对于云原生相关技术的了解和学习，对于今后的职业发展大有裨益，以下也是个人对于课程的一些开发

课程的优点：
 1. 理论+实操+作业
    每节课程的干货满满，以至于学到最后感觉自己消化的还远远不够，很多内容还值得反复咀嚼。而且整个课程并没有照本宣科，会有很多实操的环节穿插其中
    课程结束会留有相关作业，能够保证课程的参与感，所以，对于作业我是墙裂建议一定要上手去实践的
 2. 直播
    课程的形式不仅仅是录播，还有相关的直播，可以更好的与老师进行交流答疑，而且直播的内容会同步到课程中
    
课程的建议：
 1. 建议加入学生作业自评，或者每周的作业给出完成度较高的优秀作业，方便互相监督学习
 2. 最终形式的期末作业，相对契合目前公司的实际工作的实操型作业，例如给出几个应用案例，将实际的应用部署上云，能够最终供大家访问

当然短短15周的课程结束了，但是对于云原生的学习，还仅仅只是开始

```
